Phase 3 Capstone: Operating a Growing Application: When Everything You’ve Learned Is Required at Once

Week 13 Purpose (Read This First)

Weeks 10–12 intentionally created friction:

- Week 10: deployment risk and environment complexity
- Week 11: downtime and reliability failures
- Week 12: scaling and availability pain

Week 13 asks one question: Can you operate a real application responsibly with the tools you currently have?

This is not a “pass or fail” week.
It is a revelation week.

By the end, students should clearly understand:

i. what they can do
ii. what they cannot do
iii. what must be automated
iv. why orchestration becomes unavoidable


Capstone Scenario

You are responsible for operating a small but growing web application.

The application:

- is containerized using Docker
- has multiple environments
- receives increasing traffic
- experiences occasional failures
- must remain available to users

Management does not allow new tools yet.

You must operate this system using:

i. Linux
ii. Docker
iii. Docker Compose
iv. Basic CI/CD
v. Scripts
vi. Manual processes

This constraint is deliberate.

Capstone Objectives: Students must demonstrate the ability to:

- Design environment separation
- Deploy updates with minimal disruption
- Attempt scaling responsibly
- Detect and respond to failures
- Document operational decisions
- Reflect honestly on system limitations

This is engineering judgment, not memorization.
